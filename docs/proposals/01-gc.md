# WebAssembly GC Proposal

## Status: Shipped (Phase 5)

The GC extension enables efficient garbage-collected memory management for WebAssembly.

## Core Purpose

- Allocation of garbage-collected data structures
- Garbage-collected byte arrays
- Support for languages with managed memory (Java, C#, Go, OCaml, etc.)

## Design Principles

### Low-Level Data Representation
"The sole purpose of the Wasm type system is to describe low-level data layout, in order to aid the engine compiling its access efficiently."

### Key Goals
- Independence from linear memory
- Explicit runtime behavior (no implicit allocation)
- Pay-as-you-go performance model
- Support for industrial-strength garbage collectors

## Type System Extensions

### Structures
User-defined aggregates with heterogeneous, statically-indexed fields:

```wat
(type $point (struct
  (field $x f64)
  (field $y f64)
  (field $z f64)
))

(type $node (struct
  (field $value i32)
  (field $next (ref null $node))
))
```

### Arrays
Homogeneous collections with dynamic indexing:

```wat
(type $vector (array (mut f64)))
(type $bytes (array (mut i8)))
(type $strings (array (ref $string)))
```

### Reference Types

| Type | Description |
|------|-------------|
| `anyref` | Universal supertype for all GC references |
| `eqref` | References supporting equality comparison |
| `structref` | Any struct reference |
| `arrayref` | Any array reference |
| `i31ref` | Unboxed 31-bit integer |
| `funcref` | Function reference |
| `externref` | External (host) reference |

### Nullability
```wat
(ref $point)       ;; non-null reference
(ref null $point)  ;; nullable reference
```

### Unboxed Scalars (i31)
Small integers remain unboxed, avoiding heap allocation:

```wat
(ref.i31 (i32.const 42))     ;; box to i31ref
(i31.get_s (local.get $ref)) ;; unbox signed
(i31.get_u (local.get $ref)) ;; unbox unsigned
```

## Instructions

### Structure Operations

```wat
;; Allocation
(struct.new $point
  (f64.const 1.0)
  (f64.const 2.0)
  (f64.const 3.0))

(struct.new_default $point)  ;; zero-initialized

;; Field access
(struct.get $point $x (local.get $p))
(struct.set $point $x (local.get $p) (f64.const 5.0))
```

### Array Operations

```wat
;; Allocation
(array.new $vector (f64.const 0.0) (i32.const 10))  ;; 10 elements
(array.new_default $vector (i32.const 10))
(array.new_fixed $vector 3
  (f64.const 1.0)
  (f64.const 2.0)
  (f64.const 3.0))

;; Element access
(array.get $vector (local.get $arr) (i32.const 0))
(array.set $vector (local.get $arr) (i32.const 0) (f64.const 5.0))

;; Properties
(array.len (local.get $arr))

;; Bulk operations
(array.fill $vector (local.get $arr)
  (i32.const 0)    ;; offset
  (f64.const 0.0)  ;; value
  (i32.const 10))  ;; count

(array.copy $vector $vector
  (local.get $dst) (i32.const 0)
  (local.get $src) (i32.const 0)
  (i32.const 10))
```

### Type Testing and Casting

```wat
;; Runtime type test
(ref.test (ref $point) (local.get $any))  ;; returns i32

;; Cast (traps on failure)
(ref.cast (ref $point) (local.get $any))

;; Null check
(ref.is_null (local.get $ref))
(ref.as_non_null (local.get $ref))

;; Branch on cast
(br_on_cast $label (ref null any) (ref $point) (local.get $any))
(br_on_cast_fail $label (ref null any) (ref $point) (local.get $any))
```

### Extern Conversions

```wat
;; Convert between any and extern
(any.convert_extern (local.get $ext))
(extern.convert_any (local.get $any))
```

## Subtyping

### Structural Subtyping
```wat
(type $point2d (struct
  (field $x f64)
  (field $y f64)))

(type $point3d (sub $point2d (struct
  (field $x f64)
  (field $y f64)
  (field $z f64))))
```

### Final Types
```wat
(type $sealed (sub final $base (struct ...)))
```

### Recursive Types
```wat
(rec
  (type $tree (struct
    (field $value i32)
    (field $left (ref null $tree))
    (field $right (ref null $tree)))))
```

## Type Hierarchy

```
                       any
                        │
          ┌─────────────┼─────────────┐
          │             │             │
        func           eq          extern
          │      ┌──────┼──────┐      │
       nofunc  i31   struct  array  noextern
                        │
                      none
```

## Memory Management

### No Explicit Deallocation
- All GC objects are managed by the runtime
- No `free` or explicit memory management
- Collector reclaims unreachable objects

### Finalization
- No built-in finalization support
- Languages must implement their own if needed

## Use Cases

### Object-Oriented Languages (Java-like)
```wat
(type $object (struct
  (field $vtable (ref $vtable))))

(type $vtable (struct
  (field $method0 (ref $func0))
  (field $method1 (ref $func1))))
```

### Functional Languages (ML-like)
```wat
(type $closure (struct
  (field $code (ref $code_type))
  (field $env (ref $env_type))))
```

### Untyped Languages (Scheme/Python-like)
```wat
;; Tagged union approach
(type $value (struct
  (field $tag i32)
  (field $payload i64)))

;; Or use i31ref for small ints + heap for others
```

## Implementation Notes

### Engine Requirements
- Must implement tracing garbage collector
- May use any GC algorithm (mark-sweep, copying, generational, etc.)
- Must handle cycles in object graphs

### Performance Considerations
- Struct/array access should compile to direct memory operations
- Type checks can often be optimized away statically
- i31ref avoids allocation for small integers
